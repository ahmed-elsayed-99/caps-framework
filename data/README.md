# Data Schema — CAPS Framework

## Required Columns

| Column | Type | Description |
|---|---|---|
| `consumer_id` | string | Unique consumer identifier |
| `adoption_lag_days` | int | Days from launch event to first trial purchase |
| `brand_switch` | int (0/1) | 1 if prior brand expenditure fell by ≥30% within window |
| `price_elasticity` | float | Own-price elasticity estimate (negative values expected) |
| `attribute_priority_score` | float [0,1] | First factor score from attribute weighting decomposition |
| `loyalty_retention_ratio` | float [0,1] | Post-event / pre-event expenditure share for prior brand |

## Optional Columns

| Column | Type | Description |
|---|---|---|
| `category_spend_share` | float [0,1] | Category as share of total consumer spending |
| `trial_frequency` | int | Number of trial purchase occasions within window |
| `price_sensitivity_index` | float | Composite index from price response data |

## Observation Window

All behavioral signals must be computed within a consistent observation window
starting from the competitive launch event date t0. The default window is 90 days.
Minimum recommended window is 60 days. Windows below 30 days produce unstable
cluster solutions.

## Data Sources

Real applications typically draw from:
- Retail scanner panel data (Nielsen, IRI, Kantar)
- Loyalty card transaction records
- E-commerce clickstream and purchase logs
- Conjoint study outputs for attribute_priority_score

## Synthetic Sample

`raw/sample_launch_event.csv` is generated by `caps.acquisition.loaders._generate_synthetic_data`.
It models four latent purchasing power segments with realistic signal covariance
structure. Column `true_segment` is included for validation purposes only and
is not used by the pipeline.
